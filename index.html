Lab Submissions

From UI foundations to local storage

Each lab helped me build one important part of Android development, starting with interface design and gradually moving toward navigation, state management, and persistent data.

Lab 1

User Interface Design

Lab 1 introduced the foundation of building mobile screens using Jetpack Compose, including layout structure, text, buttons, spacing, and basic visual hierarchy.

Reflection: This lab helped me understand how layout structure affects the overall user experience. I learned that even a simple screen requires careful attention to alignment, spacing, readability, and consistency.

Lab 2

User Interaction & State Management

Lab 2 focused on making the application more dynamic by responding to user input and managing state changes inside Jetpack Compose.

Reflection: Through this lab, I learned how user input can change the behaviour and appearance of an application. It improved my understanding of state management and showed me how important interaction is in creating a useful mobile experience.

Lab 3

Material Design Implementation

Lab 3 explored Material Design components to improve visual consistency, usability, colours, spacing, and interface structure.

Reflection: This lab showed me how Material Design can make an application look more professional and consistent. I learned that good interface design is not only about decoration, but also about making the app clearer and easier to use.

Lab 4

Navigation & ViewModel

Lab 4 focused on creating multiple screens, managing application flow, and using ViewModel to support cleaner screen logic and data handling.

Reflection: Lab 4 helped me understand how mobile applications are structured across multiple screens. I also learned that ViewModel is useful for separating logic from the interface, making the application easier to manage and expand.

Lab 5

Room Database Integration

Lab 5 introduced Room Database for storing, retrieving, updating, and managing local data directly on the Android device.

Reflection: This lab helped me understand how applications can keep important data even after the app is closed. Learning Room Database also made me more aware of how data models, DAO, repositories, and ViewModel work together in a mobile application.

Reflection

Learning journey and growth

This section reflects on my development process from basic lab exercises to a more complete Android application.

My learning journey from Lab 1 to Project 2 showed me how mobile application development grows step by step. I started with simple user interface design in Jetpack Compose, then moved into user interaction, Material Design, navigation, ViewModel, local database storage, Firebase, API integration, and sensor-based features.

The most challenging parts were understanding navigation flow, managing data properly with ViewModel, connecting Room Database with the application structure, and integrating Firebase Firestore with external data from the OpenFoodFacts API. Sensor integration using GPS Location Services also helped me understand how mobile applications can use device hardware to create more contextual and useful features.

Through FoodLink, I learned that software engineering is not only about writing code, but also about solving meaningful problems. By focusing on SDG 2: Zero Hunger, this project helped me see how mobile technology can encourage better food management, reduce waste, and support more responsible decision-making in daily life.
