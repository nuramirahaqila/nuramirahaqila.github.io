Projects

Project 1

Project 1: FoodLink

Lab 4 + Working Features

Food waste is a common issue where usable food may be thrown away instead of being managed properly. FoodLink was developed to support SDG 2: Zero Hunger by helping users organise food-related information and encourage more responsible food usage.

Reflection: Through Project 1, I learned how to combine navigation, screen structure, and working app features into one complete mobile application. I also understood the importance of designing an app around a real SDG problem instead of only focusing on technical requirements.

Project 2

Project 2: FoodLink

Lab 5 + API + Sensors + Cloud

Many users may struggle to track food information, store food records, and connect food-related data with real-time or shared information. Project 2 upgrades FoodLink with Room Database, Firebase Firestore, API integration, and GPS sensor features to make the application more connected and useful.

Reflection: In Project 2, I learned how to integrate advanced mobile development features such as Room, Firebase Firestore, Retrofit API, and GPS sensor. This project helped me understand how modern mobile applications combine local data, cloud storage, internet data, and hardware features to solve real-world problems.
